# The Thrive Vault

The Thrive Vault is the source of truth for Thrive OS: an Obsidian vault holding the constitution, vision, architecture, decisions, specifications, projects, procedures, and AI-collaboration notes that govern the system. Implementation must follow what's documented here, not chat history.

## Structure

- `01 Constitution/` — foundational, rarely-changed principles.
- `02 Vision/` — long-term direction and goals.
- `03 System Definition/` — canonical conceptual answer to "what is Thrive OS."
- `04 Guiding Principles/` — enduring principles for judgment the Constitution doesn't explicitly settle.
- `05 Governance/` — concrete decision-making, review, and amendment processes.
- `06 Architecture/` — current, as-built system design.
- `07 ADR/` — Architecture Decision Records.
- `08 Specifications/` — approved specs that implementation must follow.
- `09 Projects/` — active and planned project notes.
- `10 SOP/` — standard operating procedures.
- `11 AI Collaboration/` — AI collaboration notes and conventions.
- `12 Templates/` — reusable note templates for the folders above.
- `99 Archive/` — retired or superseded content.

Folder numbers are contiguous and represent conceptual order, per `06 Architecture/Proposals/VAULT-IA-001 - Thrive Vault Information Architecture.md` (Ratified).

See `CLAUDE.md` at the repository root for the full governing instructions, and `CONTRIBUTING.md` for how work flows through this structure.
